Icons

Icons and icon styling

Basic Icon Usage

A majority if not all icons are displayed using inline <svg> with a wrapping <span> tag with the .icon class, the .icon--{{icon-name}} class (as a data URI fallback for lame browsers), and an optional size modifier classes.

Code Example
										
	<span class="icon icon--check-mark"><svg><use xlink:href="#icon-check-mark"></use></svg></span>

Icon Sizes

Use an alternative icon size modifier class to choose a size other than the base .icon size. Sizes range from xs-xxl. When using an icon size modifier be sure to order classes like so: .icon, .icon--{{icon-size}}, .icon--{{icon-name}}

Code Example
									 
<span class="icon icon--diamond"><svg><use xlink:href="#icon-diamond"></use></svg></span>

Input Fields

These are our default input text field styles. Here are some basic rules when writing markup for form inputs:

  1. Inputs should always be inside of a <form> element.
  2. All inputs should have a corresponding label element with matching for and id attributes for accessibility and proper functionality
  3. Each coupled label and input elements should be surrounded by the .field class

The error colored background and border will be displayed in browsers that support the :invalid pseudo class, but we'll be using a JS validator on all forms to ensure progressive enhancement and browsers will also be receiving the .is-error class on the invalid input's surround .field block. The invalid states below are seen when a user inputs an incorrect value for the specified input type.

Code Example
                    
    // Entry Field 1
    <div class="field">
        <label for="username">Username</label>
        <input type="text" id="username" placeholder="Placeholder">
    </div>

    // Entry Field 2
    <div class="field">
        <label for="username-2">Username</label>
        <input class="field__input--underline" type="text" id="username-2" placeholder="Placeholder">
    </div>

     // Error with Feedback
    <div class="field is-error">
        <label for="username-2">Username</label>
        <input type="text" id="username-2" placeholder="Placeholder">
        <span class="field__feedback">*Invalid Error Message</span>
    </div>

Select Dropdown

Use this structure for select dropdowns. Add .field__select--underline to the select element to get the alternative apperance.
Add .is-error to the .field class to get error notiication.

Code Example
                    
    // Dropdown 1
    <div class="field">
        <label for="select-1">Entry Title</label>
        <select id="select-1">
            <option disabled selected>Select Options</option>
            <option value="option-1">Option 1</option>
            <option value="option-2">Option 2</option>
            <option value="option-3">Option 3</option>
        </select>
    </div>

    // Dropdown 2
